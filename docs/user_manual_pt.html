Introdução

Esta ferramenta é voltada para usuários dos recursos computacionais do CENAPAD-UFC. Através destes portais o usuário possui a capacidade de executar programas e-Science utilizando esta interface web e os recursos computacionais de alto desempenho presentes no CENAPAD-UFC.

Para utilizar os recursos computacionais disponíveis no CENAPAD sem utilizar esta interface, o usuário deve ter conhecimento dos comandos e tecnologias de interface de texto disponíveis no cluster e nas máquinas virtuais como os comandos ssh, linux shell e comandos SLURM de manutenção do cluster.

Através desta interface espera-se que os pesquisadores que utilizam aplicações e-Science não precisem de treinamento extra para manipular as infraestruturas computacionais disponíveis, possibilitando assim uma maior objetividade dos usuários para com suas pesquisas e objetivos.

Para acessar a ferramenta é necessário requisitar as credencias de acesso ao sistema junto ao CENAPAD-UFC. A tela de login querer um usuário e senha na base de usuários do sistema.

Após apresentar as credenciais o usuário é movido para a tela principal do sistema, na qual ele tem acesso a todas aplicações organizadas em uma hierarquia de abas.

Executando Aplicações

Para o usuário utilizar os recursos disponíveis ele deve definir uma execução e submeter esta definição ao portal. O portal se encarregará de executar e disponibilizar o resultado.

4 aplicações estão disponíveis atualmente:
  • Simulações de redes computacionais com NS3
  • Simulações de dinâmicas moleculares com NAMD
  • Simulações de dockings moleculares com AutoDock
  • Execução de programas MatLab com octave

Para selecionar qual aplicação deverá ser executada o usuário deve selecionar a aba daquela aplicação na tela de portais após o login do usuário, como exibido na imagem abaixo.

Importante

Espera-se que o usuário do portal saiba utilizar as aplicações em ambientes comuns. O objetivo deste manual não é ensinar a utilizar as aplicações, o objetivo é apenas ensinar a utilizar esta interface para trabalhar com estas aplicações.

É recomendado que o usuário utilize o web site da ferramenta para aprender e tirar duvidas sobre a utilização das aplicações.

O simulador de redes computacionais, NS3

O simulador de redes NS3 é um framework C++ open source para virtualização de componentes de redes computacionais. Utilizando este framework, pesquisadores podem ter acesso ao funcionamento de diversos protocolos e componentes de rede para realizar testes de eficiência e comportamento destas entidades. Mais informações e documentação sobre o simulador NS3 podem ser encontrados aqui.

Atualmente o portal suporta simulações de rede mesh e a submissão de código de usuário que utilizem o framework padrão.

Execuções em mesh devem ser submetidas utilizando as seguintes abas:

Redes mesh em grade
Redes mesh em disco

O usuário simplesmente precisa selecionar quais as características de rede que ele deseja em sua simulação utilizando os inputs do lado direito. Não há a necessidade do usuário estudar o framework para criar código C++ para simular a topologia

O simulador de dinâmicas moleculares, NAMD

O simulador NAMD possibilita a estudar interações entre moléculas. Informações extras sobre o NAMD e manuais podem ser encontrados aqui.

O usuário deve passar os arquivos PDB, PSF e INP/XPLOR como entrada. Estes arquivos podem ser criados através de ferramentas auxiliares.

Simulações podem ser iniciadas do zero ou podem ser a partir do final de outra simulação. Para iniciar simulações a partir do resultado de outras deve-se entrar com os arquivos COOR e VEL gerados no final de outra simulação.

O modo de simulação múltipla permite dividir simulação em partes iguais para que os arquivos de resultados sejam menores e não haja problema com o programa utilizado para visualizar os resultados.

O interpretador de linguagem matemática, Octave

Octave é um interpretador para Linux que executa uma linguagem computacional parecida com a linguagem MatLab utilizada em programação cientifica. Mais informação aqui.

O usuário deve escrever o código principal do script que vai ser executado. Qualquer outro arquivo necessário para a execução do script (arquivos de dados ou bibliotecas de software) deve ser colocado no input para outros arquivos. Os parâmetros de execução do script podem ser passados utilizando o input de texto presente no portal.

Manipulação de execuções

Após uma execução ser iniciada ela fica disponível na tabela do canto inferior da tela. Através desta tabela é possível manipular cada execução para verificar status da execução, excluir e adquirir resultados.

Verificar status das execuções

A segunda coluna da esquerda pra direita da tabela apresenta o status de cada execução. A marca verde indica que a execução terminou, mas não necessariamente com sucesso; o circulo animado preto e branco indica que ainda há algo executando. A terceira coluna apresenta qual aplicação está sendo executada.

O simbolo das setas verdes no topo da tabela indica a atualização da tabela. Caso esteja girando, o portal está buscando os status de suas execuções. A cada 5 segundos o portal atualiza automaticamente o status das execuções. Isso pode ser visualizado quando as setas começam a girar automaticamente. O usuário pode iniciar uma atualização clicando nas setas.

Fazer download de resultados de execuções

Para fazer download dos arquivos gerados pela execução o usuário deve utilizar o botão da quinta coluna da esquerda para a direita, o botão com o ícone da caixa com uma seta para baixo.

Ao clicar neste botão, os arquivos gerados por aquela execução serão compactados em um arquivo zip para que o usuário faça download.
